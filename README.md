# Objective:
- To build a mobile app that forces active recall via notifications 

## Problem faced:
- Notion, Obsidian, Anki-cards have one issue. The user has to open it.
- Across multiple domains there are several key points that must be kept in mind like for Interview, gate etc

## Solution proposed:
- The product involves psychology and human behaviour
- 1 Notification every 4-5 hours from the database that forces recall

## Version 1.0 features
- Takes random questions from json files and sends the notification after atleast 15 minutes
- Send Notification button for manual trigger

## Working mechanism V1.0
- User will study a topic
- Create questions and it's answer in a local database or notebook
- Update json file manually
- This is will provide 3 layer enforcement before the app starts helping
